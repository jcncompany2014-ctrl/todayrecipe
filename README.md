# 오늘 몇 그릇?

**외식 소상공인을 위한 AI 메뉴 마진·가격결정 웹앱** — K-AI 콘텐츠 공모전 Track B

식자재를 마켓컬리처럼 장바구니에 담으면, **조리 수율(yield)까지 반영한 진짜 1인분 원가 → 적정 판매가 → 하루 몇 그릇 팔아야 본전인지**를 알려주는 모바일 웹앱.

## 차별화의 심장 — 수율(yield)

원물 단가가 아니라 *조리 후 수율을 반영한* 실제 원가를 계산합니다.
예) 앞다리살 150g을 볶으면 수율 80% → 실제 원가는 표기 단가보다 비싸집니다.

| 조리법 | 수율 |
|---|---|
| 생 | 100% |
| 삶기 | 90% |
| 볶기 | 80% |
| 튀김 | 75% |

```
재료별 실투입원가 = round( 원/g × 사용량g ÷ (수율/100) )
1인분 원가        = Σ 재료별 실투입원가 + 부대비용
마진율(%)         = round( (판매가 − 1인분 원가) ÷ 판매가 × 100 )
손익분기 그릇수   = ceil( 하루 고정비 ÷ 그릇당 남는 돈 )
```

## 화면 흐름

랜딩 → **내 메뉴판**(홈) → **식자재 마트**(담기·실시간 마진) → **장바구니**(수율·사용량 입력) → **마진 결과**(추천가·손익분기·AI 조언) · **대시보드** · **비전(Coming Soon)** · **설정**

## 기술 스택

- **React 18 + Vite** (모바일 우선 SPA, react-router)
- 상태는 인메모리(Context) — 백엔드/스토리지 불필요
- 디자인: Pretendard, 라인 SVG 아이콘(이모지 금지), 마진 신호등(초록/앰버/빨강)
- 식자재 사진: AI 생성(라이선스 안전) + SVG 폴백

## 로컬 실행

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # dist/ 프로덕션 빌드
```
