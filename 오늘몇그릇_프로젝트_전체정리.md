# 오늘 몇 그릇? — 프로젝트 전체 정리 & 개발 핸드오프

> 이 문서 하나로 처음부터 끝까지 파악하고 Claude Code에서 이어서 개발하기 위한 사양서입니다.
> 함께 첨부된 작동 HTML 3개(`내메뉴판`, `식자재마트`, `장바구니`)가 디자인 시스템과 해당 화면의 **레퍼런스 구현**입니다.

---

## 0. 한 줄 요약

**오늘 몇 그릇?** — 외식 소상공인이 식자재를 마켓컬리처럼 장바구니에 담으면, **조리 수율까지 반영한 진짜 1인분 원가 → 적정 판매가 → 하루 몇 그릇 팔아야 본전인지**를 알려주는 모바일 웹앱.

- **부제:** 외식 소상공인을 위한 AI 메뉴 마진·가격결정 웹앱
- **제출처:** K-AI 콘텐츠 공모전 / Track B 솔루션 부문 / 대학(원)·일반부
- **차별화의 심장:** **수율(yield)** — 원물 단가가 아니라 *조리 후 수율을 반영한* 실제 원가를 계산. (예: 앞다리살 150g을 볶으면 수율 80% → 실제 원가는 표기 단가보다 비싸짐)

## 1. 공모전 제약 (모든 설계의 근거)

이 제약들이 디자인·기능 결정을 강제하므로 개발 시 반드시 지켜야 함.

| 항목 | 내용 |
|---|---|
| 제출물 | **구동 가능한 웹페이지 링크(URL).** 실제로 눌리고 작동해야 함 (소개용 랜딩만 내면 안 됨) |
| 접수 기간 | 2026.7.1 ~ 7.15 17시 |
| 결과 발표 | 8월 1주 / 시상식 8월 2주 |
| 수상 후 | 9~11월 고도화 + 멘토링 필수 |
| 상금 | Track B 5팀, 각 500만원 + 통합 대상 1,000만원 |

**심사 배점 (100점) — 이게 화면 우선순위를 정함:**

| 항목 | 배점 | 우리 대응 |
|---|---|---|
| 문제해결력·창의성 | 20 | 본인이 겪은 실제 창업 원가 스트레스 → 진짜 통증 |
| 기술 구현 완성도 | 25 | **실제 작동**(담기→계산→결과). 치명적 버그 없게 |
| UX·디자인 심미성 | 25 | 컬리식 직관 장보기 + "AI 대충 느낌" 제거 |
| 유용성·발전가능성 | 30 | 사회·경제적 효용 + Coming Soon(비전)으로 확장성 |

**반드시 지킬 규칙 2가지:**
1. **라이선스:** 오픈소스·공공데이터·이미지의 출처/라이선스 준수. 위반 시 **심사 제외**. → 식자재 사진을 인터넷에서 무단으로 긁으면 안 됨. (무료 상업이미지 / AI 생성 / 고퀄 렌더 중 택1)
2. **출시 기능과 비전(Coming Soon)을 섞지 말 것.** 작동 안 하는 걸 작동하는 척 넣으면 완성도 전체가 의심받음. (자세히는 §9)

## 2. 핵심 컨셉과 우리가 내린 결정들 (왜 이렇게 만드는가)

개발 중 "왜 이렇게?"가 헷갈릴 때 참고. 이 결정들엔 다 이유가 있음.

**(1) 아이템 선정 경로**
경제·금융 → 소상공인 AI → 여러 아이디어 폐기(리뷰분석/홍보생성/지원금찾기/가계부/미수금/인수인계/상담봇) → **외식업 원가·마진 계산**으로 확정. 이유: 본인이 창업 준비하며 *원물 단가 일일이 찾고 비교하는* 실제 스트레스를 겪음 → 서사가 진짜.

**(2) "그냥 GPT한테 물으면 되잖아"를 통과하는 게 합격 기준**
공모전 아이디어 절반은 이 반박으로 죽음. 웹앱이 정당화되려면 LLM 채팅이 *못 하는* 3가지를 해야 함:
- **데이터가 쌓임** (메뉴·재료가 저장돼 누적 — GPT는 매번 리셋)
- **다단계 실행** (묻고 답이 아니라 입력→처리→산출물)
- **떠먹여주는 UI** (버튼·화면으로, 프롬프트 칠 줄 몰라도 씀)
→ 우리 앱은 셋 다 충족. (계산기/상담봇류는 탈락이라 폐기함)

**(3) UX 대전환: "마진 계산기" → "마진이 입혀진 식자재 마트"**
초기엔 단계형 입력 계산기였으나, **마켓컬리/쿠팡처럼 식자재를 장바구니에 담는 직관적 경험**으로 무게중심 이동. 담는 행위가 주인공, 마진 결과는 뒷단. 일반 마트와 다른 점 = 각 상품에 **시세 정보 + 담으면 마진이 어떻게 변하는지**가 붙음.

**(4) 2단 구조: 랜딩 → 작동하는 앱**
소개용 랜딩(그럴싸함)과 작동하는 본체(공모전 점수) 둘 다 필요 → 랜딩에서 **"체험해보기"** 누르면 작동 앱으로 진입. 비전(앱 정식 출시·도매가 연동)은 랜딩과 Coming Soon에 배치.

**(5) 디자인: "AI가 대충 만든 느낌" 제거가 최우선 미션**
첫 시안들이 평범해서 갈아엎음. 원인과 대응은 §5 참고. 핵심: **크림+테라코타(AI 클리셰 1순위) 의도적 회피, 이모지 금지, 폰트 위계 극단화.**

## 4. 디자인 시스템 (가장 중요 — 모든 화면 공통)

### 4.1 "AI 대충 느낌"을 죽이는 7원칙

1. **이모지 아이콘 절대 금지** → 통일된 라인 SVG 아이콘만 (stroke 1.6~2, round cap, 단색). 이게 싸구려 느낌의 1순위 원인이었음.
2. **크림색 쓰지 말 것** → 깨끗한 화이트. (크림 배경 + 세리프 + 테라코타 = AI가 만든 티 나는 클리셰 1순위. 의도적으로 회피.)
3. **폰트 위계 극단적으로** → 숫자(마진율·원가·그릇수)는 크고 굵게(800) + tabular, 라벨은 작고 옅게(muted).
4. **색 절제** → 잉크 + 마진 신호등(초록/앰버/빨강) + 웜뉴트럴만. 따뜻함은 *색*이 아니라 웜톤 화이트·웜 그림자·웜 아이콘 타일로.
5. **여백 넉넉하게.**
6. **시그니처는 한 곳에** → 영수증 총액 같은 "본전" 잉크 카드. 나머지는 조용하게.
7. **모션 최소** → 과한 애니메이션도 AI 느낌. 탭/스테퍼 press 피드백 + 부드러운 수치 전환 정도만.

### 4.2 컬러 토큰 (CSS 변수 그대로 사용)

```css
:root{
  /* 배경 (폰 바깥 그라데이션) */
  --bg-1:#F2EFEA; --bg-2:#E6E1DA; --bg-3:#DED8D0;
  /* 면 */
  --paper:#FFFFFF;
  /* 텍스트 */
  --ink:#1A1613; --ink-soft:#55504B; --muted:#A39B92;
  /* 라인·면 보조 */
  --line:#EDE9E3; --tile:#F5F2EE; --track:#EEEAE4;
  /* 마진 신호등 */
  --good:#16A06A;  /* 마진 ≥30% */
  --warn:#D69412;  /* 마진 20~29% */
  --bad:#D04B3F;   /* 마진 <20% */
  /* 따뜻한 액센트 배지 (수율 칩 등) */
  --sand:#F3EBE2; --sand-ink:#9A6B3C;
}
```

배경 그라데이션: `radial-gradient(125% 90% at 50% -10%, var(--bg-1), var(--bg-2) 58%, var(--bg-3))`

### 4.3 식자재 카테고리별 사진 톤 (플레이스홀더 + 추후 실사진 톤 기준)

저채도 = 사진처럼 보이고 클리셰 회피. 카테고리 = 색 코딩 역할도 함.

| 카테고리 | 배경 그라데이션 | 아이콘 색 |
|---|---|---|
| 정육 meat | `linear-gradient(140deg,#F7E8E4,#EFD8D2)` | `#BC857B` |
| 청과 veg | `linear-gradient(140deg,#E9F0E3,#DAE7D1)` | `#85A06E` |
| 양념 sauce | `linear-gradient(140deg,#F5EDDE,#ECE0C9)` | `#B89767` |
| 수산 sea | `linear-gradient(140deg,#E5ECF0,#D6E1E7)` | `#7F98A7` |
| 기타 etc | `linear-gradient(140deg,#F5F0E7,#ECE4D6)` | `#AF9F85` |

> **사진 처리:** 현재는 위 톤 + 라인 아이콘 플레이스홀더. 레이아웃 확정됨 → 실사진으로 교체 시 **라이선스 안전한 소스만**(무료 상업이미지 / AI 생성 / 고퀄 렌더). 톤은 위 카테고리 색감에 맞추면 통일감 유지.

### 4.4 타이포

- **폰트:** Pretendard Variable (CDN: `https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable.min.css`)
- 패밀리: `'Pretendard Variable', Pretendard, -apple-system, ...`
- 모든 숫자: `font-variant-numeric: tabular-nums`
- 위계 예시(390px 폰 기준): 페이지 타이틀 21~25/800 · 히어로 큰 숫자 38~54/800 · 카드 제목 15~17/700 · 마진율 20~38/800 · 라벨/캡션 11~13/500~600 muted

### 4.5 아이콘 세트 (직접 그린 인라인 SVG — 이모지 금지)

24 viewBox, stroke 1.6~2, round, 단색. 작동 HTML에 SVG path 전부 들어있으니 **거기서 복사**.

- **식자재:** meat(고기 덩어리), chicken(닭다리), onion(양파), scallion(대파), garlic(마늘), egg(계란), jar(고추장 등 통), bottle(간장·기름), rice(쌀밥)
- **UI:** back(←), chevron(▾/›), search(돋보기), plus/minus(+/−), check(✓), cart(장바구니), trash/x(빼기), bars(대시보드), star/sparkle(비전), sliders(설정), trend triangle(시세 ▲▼)

### 4.6 기타 토큰

- **디바이스 프레임:** iOS 스타일. 폭 392px(데스크탑) / 모바일은 전체. radius 54(프레임)·43(화면). 상단 상태바(9:41 + 신호/와이파이/배터리 SVG).
- **그림자:** 웜틴트 + 레이어드. 예) 카드 `0 1px 2px rgba(60,40,25,.035), 0 8px 18px -9px rgba(60,40,25,.08)`. FAB/바 더 진하게.
- **모서리:** 카드 18~20, 칩 11~14, 타일 13~16.
- **인터랙션:** press 시 `scale(.9~.985)`, 수치 변화 시 `transition .14~.3s cubic-bezier(.2,.8,.3,1)`.
- **접근성/품질 바닥선:** `:focus-visible` 보이게, `prefers-reduced-motion` 존중, 모바일에서 폰이 화면 꽉 차게(미디어쿼리), 브라우저 스토리지 사용 금지(상태는 메모리/리액트 state).

## 5. 계산 로직 (정확히 이대로 구현)

### 5.1 수율 (yield)

| 조리법 | 수율 |
|---|---|
| 생 | 100% |
| 삶기 | 90% |
| 볶기 | 80% |
| 튀김 | 75% |

- 조리 안 하는 재료(양념·쌀 등)는 `cookable=false` → 수율 100% 고정, 조리탭 대신 "생 그대로" 표시.

### 5.2 원가 / 마진 / 손익분기

```
재료별 실투입원가 = round( (원/g × 사용량g) ÷ (수율/100) )
식자재 원가      = Σ 재료별 실투입원가
1인분 원가       = 식자재 원가 + 부대비용
남는 돈          = 판매가 − 1인분 원가
마진율(%)        = round( 남는 돈 ÷ 판매가 × 100 )
손익분기 그릇수  = ceil( 하루 고정비 ÷ 그릇당 남는 돈 )
```

- **마진 신호등:** ≥30% 초록 / 20~29% 앰버 / <20% 빨강
- **부대비용(데모 고정 2,750원):** 배달수수료(판매가×12%=1,080) + 포장비(300) + 인건비(880) + 가스·부자재(490). → 추후 입력화(비전 아님, 출시 개선 가능).
- **손익분기 데모값:** 66그릇(하루 고정비 기준 고정). → 추후 고정비 입력 시 자동 산출.
- 화면 표시는 **10원 단위 반올림** 권장(`round(n/10)*10`).

### 5.3 시세 트렌드 (식자재 마트)

재료별 "평소 대비" 가격 변동을 칩으로 표시: 저렴(초록 ▼) / 비쌈(빨강 ▲) / 보통(회색). **데모는 시드값.** 비전에서 실시간 도매가로 대체.

## 6. 데이터 모델 / 시드 데이터

### 6.1 모델

```ts
// 식자재 (카탈로그)
Product = {
  id, nm,                      // 'apdari', '앞다리살'
  cat: 'meat'|'veg'|'sauce'|'sea'|'etc',
  icon,                        // 아이콘 키
  price, unit,                 // 9000, '/kg'
  perG,                        // 원/g (9)
  trend: ['dn'|'up'|'fl', n],  // 시세 ['dn', 1]
  popular: boolean
}

// 장바구니 항목 (특정 메뉴에 담긴 재료)
CartItem = { productId, grams, cookable, method:'생'|'볶기'|'삶기'|'튀김' }

// 메뉴
Menu = {
  id, nm, price,               // 판매가
  ingredients: CartItem[],
  // 파생값(계산): foodCost, totalCost, profit, margin
}
```

> **백엔드 불필요(출시 데모):** 시드 데이터 + 인메모리/클라이언트 state로 충분. 공공시세 API(KAMIS)·도매가 연동은 **비전**. 배포만 하면 됨(Vercel 등).

### 6.2 시드: 식자재 마트 상품

| id | 이름 | 카테고리 | 아이콘 | 가격 | 단위 | perG | 시세 | 인기 |

## 7. 화면별 상세 명세

### 7.1 랜딩페이지 — [미구현]

**목적:** 첫인상 + 작동 앱으로의 진입. MONKi류 참고하되 **밝은 톤**(우리 디자인 시스템).

**구성(위→아래):**
1. **히어로** — "오늘 몇 그릇?" + 한 줄("감으로 정하던 메뉴 가격, 숫자로") + **[체험해보기]** 버튼.
2. **문제 공감** — "원물 단가 일일이 찾고, AI한테 물어도 추정가라 애매하고…"
3. **차별점(수율)** — 왜 우리가 다른지(조리 수율 반영한 진짜 원가).
4. **성과/신뢰** (선택) — 가벼운 지표/문구.
5. **Coming Soon** — 비전 3종(§9).

**체험 진입 방식:** [체험해보기] → 작동 앱(식자재 마트 또는 메뉴판). **권장 = 풀스크린 전환**(폰 프레임 유지 X, 모바일 대응·구현 깔끔). *※ 이건 최종 미확정 — 확정 필요.* 핵심 원칙: 폰 목업 안이 **스크린샷이면 안 되고 실제 작동**이어야 함.

### 7.2 식자재 마트 — [구현됨] `오늘몇그릇_식자재마트.html`

**목적:** 마켓컬리식 장보기. 담을 때마다 마진이 보임. 앱의 첫인상이자 심장.

**구성:**
- 헤더: 뒤로 + "재료 담기" / **메뉴 칩** "제육덮밥 담는 중 ▾" / 검색바
- **카테고리 가로 스크롤(sticky)**: 전체·정육·수산·청과·양념·기타
- **가로 스크롤 섹션** "사장님이 많이 담아요" — 인기 식자재 카드(사진+가격+담기 버튼)
- **세로 리스트** 카테고리별(정육·청과…): [사진] 이름·가격 / **시세 칩**(▼8% 저렴 등) / 담기(+) 버튼
- **하단 떠 있는 장바구니 바**: 담은 개수 + 식자재 원가 + **예상 마진**(실시간) → 장바구니로

**인터랙션(작동):**
- 담기(+) 토글 → 같은 id 모든 버튼 동기화(담음 ✓) → 하단 바 개수·원가·**예상 마진 실시간 갱신** + 토스트 "삼겹살 담음 · 예상 마진 52% → 50%"
- 예상 마진(마트) = `(판매가 − 식자재합 − 부대비용) ÷ 판매가`. 재료 적을 땐 높고, 다 담을수록 최종값(장바구니)으로 좁혀짐.

### 7.3 장바구니 — [구현됨] `오늘몇그릇_장바구니.html`

**목적:** **수율이 실제로 작동하는 유일한 자리.** 구조 = "A 입력 중심 + B 실시간 마진 살짝".

**구성:**
- 헤더: 뒤로 + "장바구니" / "제육덮밥 담는 중" 서브
- **(B) 상단 실시간 마진 카드**: 판매가 / **예상 마진 41%**(신호등 색, 큰 숫자) / 1인분 원가 / 남는 돈 / 막대바. 입력 바꿀 때마다 반응.
- **(A) 재료별 입력 카드**(메인): 각 재료에 [사진][이름·원/g] + **수율 칩** + 실투입원가 / **사용량 스테퍼**(−10/+10 g) / **조리법 탭**(생·볶기·삶기·튀김 → 수율 자동) — 양념·쌀은 "생 그대로 · 수율 100%"
- "재료 더 담기"(마트로) / **부대비용 카드**(펼치면 배달수수료·포장·인건비 내역)
- 하단 CTA "마진 확정하기 → 1인분 ₩5,310"

**인터랙션(작동):** 사용량 ± 또는 조리법 변경 → 해당 재료 실원가 재계산(살짝 bump) → **상단 마진/원가/남는돈/막대 전부 실시간 갱신.** 부대비용 펼치기. 재료 빼기.

### 7.4 마진 결과 — [미구현] (다음 차례)

**목적:** 흐름의 피날레. "마진 확정하기" 누르면 진입.

**구성:**
- **큰 숫자 3종 세트**(영수증처럼 또렷, 각 카드):
  - `1그릇 실원가 ₩5,310`
  - `9,000원에 팔면 → 1그릇당 ₩3,690 남음 (마진 41%)` (신호등 색)
  - `오늘 본전 → 하루 66그릇`
- **AI 조언 카드**(사람 말투 1~3장): 예) "마진이 빠듯해요. 9,500원으로 올리면 46%로 건강해집니다." / "배달 수수료가 원가의 20%를 먹고 있어요 — 포장 유도가 답일 수 있어요."
- **수율 설명 한 줄**(차별화 보이는 자리, 필수): "앞다리살은 볶으면 80%만 남아서, 표기 단가보다 실제 원가가 높아요."
- **가격 조정 슬라이더**: 판매가를 드래그하면 마진·손익분기 그릇 수가 실시간으로 변함.

**기본값(제육덮밥):** 원가 5,310 / 판매가 9,000 / 남는 돈 3,690 / 마진 41% / 손익분기 66그릇.

### 7.5 내 메뉴판(홈) — [구현됨] `오늘몇그릇_내메뉴판.html`

**목적:** 등록 메뉴의 마진 건강 상태 한눈에. 앱 홈.

**구성:** 헤더(브랜드+「내 메뉴판」+메뉴 N개) / **본전 잉크 카드(시그니처)** "66그릇 팔면 돼요"+점선(영수증 총액선)+평균 남는 돈·효자 메뉴 / **메뉴 카드 리스트**(아이콘·이름·판매가·마진 신호등·마진율·진행바) / 하단 탭(메뉴판·대시보드·**＋FAB**·비전·설정). 메뉴 개수·평균·효자는 자동 계산.

### 7.6 대시보드 — [미구현]

가게 전체 관점: 메뉴 마진 순위(효자↔적자), 월 고정비 입력 → 가게 손익분기 그릇 수, 간단 시각화(막대/도넛) 1개. 과하지 않게.

## 8. 출시 기능 vs 비전 (절대 섞지 말 것)

| 구분 | 내용 | 화면에서 |
|---|---|---|
| **출시 (진짜 작동)** | 수율 반영 실원가 계산 / 추천가·인상폭 / 손익분기 그릇 수 / 공공시세(또는 시드) 참고값 / AI 조언 / 메뉴별 마진 신호등 / 컬리식 장보기 | 실제로 눌리고 계산됨 |
| **비전 (Coming Soon에만)** | ① 실시간 도매가 **제휴** 연동(미트박스 등, 회원가입 없이 비교·바로구매) ② POS·배달앱 매출 연동("오늘 실제 몇 그릇 팔렸나" 자동 추적) ③ 시세 변동 알림("삼겹살 12%↑, 마진 재점검") | 별도 페이지 / "출시 예정" 배지. 작동하는 척 X |

**지원서 팁:** 발전가능성 항목에 "현재는 입력+공공시세, 향후 도매 플랫폼과 **제휴를 통한** 실시간 단가 연동으로 확장" — "제휴를 통한"이 핵심(무단 크롤링 아님을 시그널).

## 9. 진행 현황 & 다음 빌드 순서

**완료(작동 코드):** 식자재 마트 / 장바구니 / 내 메뉴판 + **디자인 시스템 확정.**

**남은 작업 권장 순서:**
1. **마진 결과 화면** (흐름 피날레, §7.4) — 마트→장바구니→결과 흐름 완성.
2. **랜딩페이지** (§7.1) — 첫인상 + 체험 진입. *체험 진입 방식 확정 필요.*

4. **대시보드** (§7.6) — 여유 있으면.
5. **화면 간 실제 라우팅 연결** — 지금은 화면별 토스트로 "다음 단계" 안내 중. 전부 이으면 끝.
6. **실사진 교체**(라이선스 안전 소스) + **배포**(작동 URL).

**Claude Code 작업 가이드:**
- 모바일 우선 SPA. §4 토큰을 CSS 변수로 그대로. 화면은 뷰/라우트로.

- 아이콘 SVG·계산 로직·시드 데이터는 **첨부 HTML 3개에서 복사**하면 100% 일치.
- 공모전 제출 = 배포된 URL. 제출 전 정상 구동 필수 확인.
